## Overview

Build a two-wheeled robot that uses an ultrasonic sensor to detect obstacles and navigate around them. This is a great first robotics project.

## Assembly

1. Mount the two DC motors into the chassis frame
2. Attach wheels to the motor shafts
3. Secure the Arduino board to the chassis with standoffs or zip ties
4. Mount the L298N motor driver board
5. Attach the ultrasonic sensor to the front of the chassis facing forward

## Wiring

### Motor Driver (L298N)

- **ENA** → Arduino pin 5 (PWM speed control)
- **IN1** → Arduino pin 6
- **IN2** → Arduino pin 7
- **IN3** → Arduino pin 8
- **IN4** → Arduino pin 9
- **ENB** → Arduino pin 10
- **12V** → Battery pack positive
- **GND** → Battery pack negative + Arduino GND

### Ultrasonic Sensor (HC-SR04)

- **VCC** → Arduino 5V
- **GND** → Arduino GND
- **TRIG** → Arduino pin 3
- **ECHO** → Arduino pin 4

## The Science

The HC-SR04 ultrasonic sensor sends a 40kHz sound pulse and measures the time for the echo to return. Distance is calculated as:

`distance = (echo_time × speed_of_sound) / 2`

At room temperature, sound travels at approximately 343 m/s, so each microsecond of echo time represents about 0.017 cm of distance.

## Code

```cpp
#define TRIG 3
#define ECHO 4

long getDistance() {
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG, LOW);
  return pulseIn(ECHO, HIGH) * 0.034 / 2;
}

void setup() {
  Serial.begin(9600);
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  // Motor pins setup...
}

void loop() {
  long distance = getDistance();
  if (distance < 20) {
    // Obstacle detected — turn right
  } else {
    // Move forward
  }
  delay(100);
}
```

## Troubleshooting

- **Robot spins in circles**: Check motor wiring polarity — swap the two wires on one motor
- **Sensor always reads 0**: Verify TRIG and ECHO pin connections
- **Motors won't spin**: Check battery voltage and L298N jumper settings
- **Robot moves erratically**: Ensure batteries are fresh — low voltage causes unpredictable motor behavior

## Test Procedure

1. Power on and verify ultrasonic sensor readings in Serial Monitor
2. Place your hand 10 cm in front of the sensor — reading should be approximately 10
3. Set the robot on the floor and verify it drives forward
4. Place an obstacle in its path — it should turn and continue
