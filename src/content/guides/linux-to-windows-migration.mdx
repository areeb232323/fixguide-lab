## Pre-flight Checklist

- [ ] Back up your entire home directory (`/home/username/`)
- [ ] Export application configurations and dotfiles
- [ ] Download the Windows ISO from [microsoft.com/software-download](https://www.microsoft.com/en-us/software-download/windows11)
- [ ] Have your Windows product key or Microsoft account ready
- [ ] Download network/Wi-Fi drivers for your specific hardware model onto a separate USB
- [ ] Have a USB drive with at least 8 GB capacity for the Windows installer

> **DATA LOSS WARNING**: Installing Windows will overwrite your Linux partition. There is no way to recover Linux data after this step unless you have a backup.

## Step 1: Backing Up Linux Data

### Essential Files

```bash
# Create a compressed backup of your home directory
tar -czf ~/backup-home.tar.gz -C /home/ $(whoami)/

# Copy to external drive (replace /media/usb with your mount point)
cp ~/backup-home.tar.gz /media/usb/
```

### Often-Forgotten Items

| What | Location | Why It Matters |
|---|---|---|
| SSH keys | `~/.ssh/` | Needed for GitHub, servers — cannot be regenerated |
| Git config | `~/.gitconfig` | Username, email, aliases |
| GPG keys | `~/.gnupg/` | Used for signing commits, encrypting files |
| Browser profile | `~/.mozilla/` or `~/.config/google-chrome/` | Bookmarks, passwords, extensions |
| App configs | `~/.config/` | Settings for VS Code, terminals, etc. |
| Crontab | Run `crontab -l > ~/crontab-backup.txt` | Scheduled tasks |
| Databases | `/var/lib/mysql/` or `/var/lib/postgresql/` | If you run local databases |

### Export Installed Packages List

Save a list of everything you have installed, in case you need to find Windows equivalents:

**Debian/Ubuntu:**
```bash
dpkg --get-selections > ~/installed-packages.txt
```

**Fedora:**
```bash
dnf list installed > ~/installed-packages.txt
```

### Verify Your Backup

**This step is mandatory.** Mount your backup drive on another machine or in a live session and verify you can open key files:

```bash
tar -tzf /media/usb/backup-home.tar.gz | head -20  # List contents
```

## Step 2: Downloading Drivers

Windows may not have drivers for your Wi-Fi adapter or GPU out of the box. Download them BEFORE wiping Linux.

### Find Your Hardware

```bash
# Network adapter
lspci | grep -i network

# GPU
lspci | grep -i vga

# Full hardware report
sudo lshw -short
```

## Step 3: Creating the Windows Installer USB

### From Another Windows PC

1. Download the [Media Creation Tool](https://www.microsoft.com/en-us/software-download/windows11) from Microsoft
2. Run it and select **"Create installation media for another PC"**
3. Choose language, edition (Windows 11 Home or Pro), and architecture (64-bit)
4. Select **USB flash drive**
5. Wait for the download and writing to complete

### From Linux

Use **WoeUSB** or **Ventoy**:

**Ventoy (recommended):**
1. Download from [github.com/ventoy/Ventoy/releases](https://github.com/ventoy/Ventoy/releases)
2. Run `sudo ./Ventoy2Disk.sh -i /dev/sdX` (replace sdX with your USB drive)
3. Copy the Windows ISO file directly onto the Ventoy USB
4. Boot from the USB — Ventoy will list the ISO for installation

**WoeUSB:**
```bash
# Install
sudo apt install wimtools
# Or use the GUI version from github.com/WoeUSB/WoeUSB-ng
pip3 install WoeUSB-ng

# Create bootable USB
sudo woeusb --device /path/to/windows.iso /dev/sdX
```

Reference: [Ventoy GitHub](https://github.com/ventoy/Ventoy) | [WoeUSB-ng GitHub](https://github.com/WoeUSB/WoeUSB-ng)

## Step 4: Installing Windows

1. Insert the Windows USB and restart your computer
2. Enter the boot menu (F12, F9, F2, or Esc depending on manufacturer)
3. Select the USB drive as the boot device
4. Select your language and click **"Install now"**
5. Enter your product key or click **"I don't have a product key"** (you can activate later)
6. Choose your Windows edition (Home or Pro)
7. Select **"Custom: Install Windows only (advanced)"**
8. **Delete the Linux partitions:**
   - Select each Linux partition (usually ext4 and swap) and click **Delete**
   - Select the resulting **Unallocated space** and click **Next**
   - Windows will create the necessary partitions automatically
9. Wait for installation to complete (20–40 minutes)

> **CRITICAL**: If you have multiple drives, make absolutely sure you are deleting partitions on the correct drive. Check the drive size to confirm. Disconnect other drives if unsure.

### Removing GRUB

After Windows installs, the GRUB bootloader entries may remain in UEFI. Windows usually overwrites them, but if you see GRUB remnants:

1. Boot into Windows
2. Open **Command Prompt as Administrator**
3. Run:
```cmd
bcdedit /enum firmware
:: Find the Linux entry and note its identifier
bcdedit /delete {identifier}
```

Or use [EasyUEFI](https://www.easyuefi.com/) for a GUI approach.

## Step 5: Post-Install Setup

### Priority Order

1. **Network drivers**: Install from your driver USB if Wi-Fi doesn't work out of the box
2. **Windows Update**: Run Settings → Windows Update → Check for updates. Repeat until no more updates appear
3. **Chipset drivers**: Download from your laptop/motherboard manufacturer's support page
4. **GPU drivers**:
   - NVIDIA: [geforce.com/drivers](https://www.nvidia.com/en-us/geforce/drivers/)
   - AMD: [amd.com/en/support](https://www.amd.com/en/support)
   - Intel: Windows Update usually handles Intel graphics
5. **Restore your data**: Copy files from your backup drive to the appropriate Windows folders

### Recommended Software Setup

| Category | Recommended | Source |
|---|---|---|
| Browser | Firefox / Chrome | [mozilla.org](https://www.mozilla.org/firefox/) / [google.com/chrome](https://www.google.com/chrome/) |
| Code Editor | VS Code | [code.visualstudio.com](https://code.visualstudio.com/) |
| Terminal | Windows Terminal | Pre-installed on Windows 11, or [GitHub releases](https://github.com/microsoft/terminal/releases) |

| Package Manager | winget (built-in) or Chocolatey | [chocolatey.org](https://chocolatey.org/) |
| Archive Tool | 7-Zip | [7-zip.org](https://www.7-zip.org/) |

**Bulk install with winget:**
```powershell
winget install Mozilla.Firefox
winget install Microsoft.VisualStudioCode
winget install Git.Git
winget install 7zip.7zip
```

### WSL (Windows Subsystem for Linux)

If you still need Linux tools, install WSL instead of dual-booting:

```powershell
wsl --install
```

This gives you a full Linux terminal inside Windows. Reference: [Microsoft WSL Docs](https://learn.microsoft.com/en-us/windows/wsl/install)

## Common Mistakes

- Forgetting hidden config files (dotfiles starting with `.`)
- Not having network drivers ready offline for the initial setup
- Skipping the verification of the backup before wiping Linux
- Losing SSH keys and having to regenerate/re-authorize them everywhere
- Not running Windows Update fully before installing other software
- Deleting partitions on the wrong drive

## Verification

- [ ] Windows boots and runs correctly
- [ ] Network connectivity works (Wi-Fi and/or Ethernet)
- [ ] All backed-up files are accessible and intact
- [ ] Key applications are installed
- [ ] Device Manager shows no missing drivers (no yellow warning icons)
- [ ] Windows is activated (Settings → System → Activation)
