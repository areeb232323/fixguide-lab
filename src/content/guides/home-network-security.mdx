## Pre-flight Checklist

- [ ] Know your router's admin IP address (usually 192.168.0.1 or 192.168.1.1)
- [ ] Have your router admin username and password (check the sticker on the router or your ISP paperwork)
- [ ] Have a device connected to the network (wired or wireless)
- [ ] Note your current Wi-Fi password — you'll need it if settings reset

> **NETWORK DISRUPTION**: Changing router settings may temporarily disconnect all devices. Do this when others in your household are aware. Never change settings on a network you don't own.

## Step 1: Access Your Router Admin Panel

1. Open a browser and go to your router's IP:
   - Most routers: `192.168.0.1` or `192.168.1.1`
   - To find yours, open Command Prompt/Terminal and run:
     - **Windows**: `ipconfig` → look for "Default Gateway"
     - **macOS/Linux**: `ip route | grep default` or `route -n get default`
2. Log in with admin credentials
3. If you've never changed the password, check the router's label or search your model at [routerpasswords.com](https://www.routerpasswords.com/)

Reference: [r/HomeNetworking — Router Setup Guide](https://www.reddit.com/r/HomeNetworking/wiki/index)

## Step 2: Change Default Admin Password

The #1 security vulnerability is leaving the default admin password:

1. Navigate to **Administration** or **System** settings
2. Change the admin password to something strong (12+ characters, mix of letters/numbers/symbols)
3. Save the new password in a password manager like [Bitwarden](https://bitwarden.com/) (free, open-source) or [1Password](https://1password.com/)

## Step 3: Update Router Firmware

Outdated firmware is a major security risk:

1. Find your router's model number (on the label)
2. Visit the manufacturer's support page:
   - [TP-Link](https://www.tp-link.com/us/support/download/)
   - [NETGEAR](https://www.netgear.com/support/)
   - [ASUS](https://www.asus.com/support/)
   - [Linksys](https://www.linksys.com/support/)
   - [D-Link](https://support.dlink.com/)
3. Download the latest firmware for your exact model
4. In the router admin, go to **Firmware Update** or **System Update**
5. Upload the firmware file and wait for the update to complete

> **DO NOT** unplug the router during a firmware update. A failed update can brick the device.

**Advanced option**: Flash [OpenWrt](https://openwrt.org/) or [DD-WRT](https://dd-wrt.com/) firmware for more control. Check compatibility first: [OpenWrt Table of Hardware](https://openwrt.org/toh/start)

## Step 4: Secure Wi-Fi Settings

### Use WPA3 (or WPA2 if WPA3 is unavailable)

1. Go to **Wireless Settings** → **Security**
2. Set encryption to **WPA3-Personal** (best) or **WPA2-Personal (AES)** (minimum)
3. **Never use**: WEP (cracked in minutes), WPA-TKIP (deprecated), or "Open" (no encryption)

### Set a Strong Wi-Fi Password

- Minimum 12 characters
- Mix of uppercase, lowercase, numbers, and symbols
- Do NOT use your address, name, or common words

### Change the Default SSID

1. Don't broadcast your router model in the network name (e.g., "NETGEAR-5G" tells attackers your router model)
2. Use something generic that doesn't identify you or your address

Reference: [CISA — Securing Your Home Network](https://www.cisa.gov/news-events/news/securing-your-home-network)

## Step 5: Set Up a Guest Network

Keep visitors off your main network:

1. Go to **Guest Network** settings (most modern routers have this)
2. Enable the guest network with a different password
3. **Disable** "Allow guests to access local network" or "Access to Intranet"
4. Set a bandwidth limit if your router supports it

This prevents guests from accessing your NAS, printers, smart home devices, or other computers.

## Step 6: Disable Unnecessary Features

Turn off features you don't use — each one is a potential attack surface:

| Feature | Recommendation | Why |
|---|---|---|
| **WPS** (Wi-Fi Protected Setup) | **Disable** | Brute-forceable in hours with [Reaver](https://github.com/t6x/reaver-wps-fork-t6x) |
| **UPnP** | **Disable** | Allows any device to open ports without your knowledge |
| **Remote Management** | **Disable** | Exposes your router admin panel to the internet |
| **Telnet/SSH access** | **Disable** (unless you use it) | Command-line access to the router |
| **WAN Ping** | **Disable** | Don't respond to internet pings |

## Step 7: DNS-Level Ad Blocking (Pi-hole)

Block ads, trackers, and malicious domains at the network level with [Pi-hole](https://pi-hole.net/):

1. Install on a Raspberry Pi or any Linux machine (see our [Pi-hole Network Ad Blocker project](/projects/pihole-network-adblocker))
2. Set your router's DNS server to the Pi-hole's IP address
3. Every device on your network is now protected — no browser extensions needed

**Alternatives if you don't want to run Pi-hole:**
- [NextDNS](https://nextdns.io/) — Cloud-based, free tier available
- [Cloudflare 1.1.1.1 for Families](https://1.1.1.1/family/) — Blocks malware (1.1.1.2) or malware+adult content (1.1.1.3)
- [Quad9](https://www.quad9.net/) — Free, blocks malicious domains

### Set Custom DNS on Your Router

1. Go to **WAN** or **Internet** settings → **DNS**
2. Replace your ISP's DNS with a secure option:
   - Cloudflare: `1.1.1.1` and `1.0.0.1`
   - Google: `8.8.8.8` and `8.8.4.4`
   - Quad9: `9.9.9.9` and `149.112.112.112`
3. Save and restart the router

## Step 8: Secure IoT Devices

Smart home devices (cameras, lights, speakers) are often the weakest link:

1. Put IoT devices on your **guest network** or a separate VLAN
2. Change default passwords on every device
3. Disable cloud features you don't use
4. Check for firmware updates regularly
5. Disable UPnP (see Step 6)

Reference: [NIST — Securing IoT Devices](https://csrc.nist.gov/publications/detail/nistir/8259/final)

## Troubleshooting

| Problem | Solution |
|---|---|
| Locked out of router | Factory reset (hold reset button 10s), reconfigure from scratch |
| Devices can't connect after changes | Forget the network on each device and reconnect with the new password |
| Pi-hole blocking legitimate sites | Add domains to the Pi-hole whitelist via the admin dashboard |
| Slow Wi-Fi after firmware update | Reset network settings and reconfigure channels (try 1, 6, or 11 for 2.4 GHz) |

## Verification

- [ ] Router admin password changed from default
- [ ] Router firmware is up to date
- [ ] Wi-Fi uses WPA2 or WPA3 with a strong password
- [ ] Guest network enabled with no local access
- [ ] WPS disabled
- [ ] UPnP disabled
- [ ] Custom DNS configured
- [ ] All IoT devices have changed default passwords
