## Pre-flight Checklist

- [ ] Identify your motherboard or laptop manufacturer and exact model number
- [ ] Have access to a second device (phone or another computer) for looking up documentation
- [ ] Ensure stable power supply — plug in the charger and/or UPS
- [ ] If on a laptop, connect the charger — **never do firmware operations on battery**
- [ ] Locate your motherboard manual (check manufacturer's support site)

> **FIRMWARE WARNING**: Modifying BIOS/UEFI settings incorrectly can prevent your computer from booting. Interrupting a firmware update can **permanently brick** your motherboard. Always ensure stable power and follow manufacturer instructions exactly.

## When to Reset BIOS/UEFI

Common symptoms that a BIOS/UEFI reset can fix:

| Symptom | Likely Cause |
|---|---|
| Computer won't boot, shows black screen | Wrong boot order or disabled boot device |
| Boot loops or restarts immediately | Unstable overclocking settings |
| "No bootable device" error | Boot order changed or Secure Boot blocking OS |
| Windows/Linux won't load after settings change | CSM/Legacy mode mismatch |
| Hardware not detected (RAM, GPU, drives) | Incorrect firmware settings |
| Fans spin but no display output | CPU overclocking failure or GPU settings |

## UEFI Entry Keys by Manufacturer

The key to press varies by manufacturer. Press it repeatedly right after powering on:

| Manufacturer | UEFI Key | Boot Menu Key | Support Page |
|---|---|---|---|
| **Dell** | F2 | F12 | [dell.com/support](https://www.dell.com/support/home) |
| **HP** | F10 or Esc | F9 | [support.hp.com](https://support.hp.com/) |
| **Lenovo** | F1 or F2 | F12 | [support.lenovo.com](https://support.lenovo.com/) |
| **ASUS** | F2 or Del | F8 | [asus.com/support](https://www.asus.com/support/) |
| **MSI** | Del | F11 | [msi.com/support](https://www.msi.com/support) |
| **Acer** | F2 or Del | F12 | [acer.com/support](https://www.acer.com/support) |
| **Gigabyte** | Del | F12 | [gigabyte.com/support](https://www.gigabyte.com/Support) |
| **ASRock** | F2 or Del | F11 | [asrock.com/support](https://www.asrock.com/support/index.asp) |

**If you can't enter UEFI at all**, see Method 2 (CMOS battery) or Method 3 (jumper reset) below.

### Entering UEFI from Windows 11

If your system boots too fast to catch the key:

1. Open **Settings** → **System** → **Recovery**
2. Click **"Restart now"** under Advanced startup
3. Select **Troubleshoot** → **Advanced options** → **UEFI Firmware Settings**
4. Click **Restart**

Reference: [Microsoft — UEFI Firmware Settings](https://support.microsoft.com/en-us/windows/start-your-pc-in-uefi-mode)

## Method 1: Reset via UEFI Menu

This is the safest and easiest method. Use this if you can still access the UEFI settings screen.

1. Restart and enter UEFI settings using the key from the table above
2. Navigate to the **Exit** or **Save & Exit** tab
3. Select **"Load Optimized Defaults"**, **"Load Setup Defaults"**, or **"Reset to Default"**
   - The exact wording varies by manufacturer
4. Confirm when prompted
5. Select **"Save Changes and Exit"**
6. Your system should boot normally with factory BIOS settings

> **Note**: This resets ALL settings including boot order, clock speed, fan curves, and Secure Boot. You may need to reconfigure your boot order afterward.

## Method 2: CMOS Battery Reset

Use this if you **cannot access the UEFI menu at all** (black screen, boot loop, no display output).

### Desktop PCs

1. **Power off** the computer completely and **unplug the power cable**
2. Press and hold the power button for 10 seconds to discharge residual power
3. Open the side panel of the case
4. **Ground yourself**: Touch the metal case frame before handling components
5. Locate the **CMOS battery** — a silver CR2032 coin-cell battery on the motherboard
6. Gently push the metal retention clip to release the battery
7. Remove the battery and **wait 60 seconds** (some boards need up to 5 minutes)
8. Reinsert the battery with the **+ side facing up**
9. Close the case, plug in the power cable, and power on
10. Enter UEFI settings and reconfigure your boot order

> **ELECTRICAL SAFETY**: Always unplug the computer and discharge static before touching internal components. Wear an anti-static wrist strap if available, or touch the metal case frequently.

### Laptops

Laptop CMOS batteries are harder to access and vary by model:

1. Check your laptop's service manual on the manufacturer's support page
2. Some laptops have the CMOS battery under a small access panel on the bottom
3. Others require full disassembly — if you're not comfortable, use Method 1 or take it to a technician
4. Search **"[your laptop model] CMOS reset"** on [iFixit](https://www.ifixit.com/) for step-by-step teardown photos

Reference: [iFixit Laptop Repair Guides](https://www.ifixit.com/Device/Laptop)

### Buying a Replacement Battery

CMOS batteries are standard **CR2032** coin cells. Available at any electronics store or online:
- [Amazon — CR2032](https://www.amazon.com/s?k=CR2032+battery)
- Any pharmacy or electronics retailer

Replace the battery if your computer keeps losing its date/time settings — that's a sign the CMOS battery is dead.

## Method 3: BIOS Recovery Jumper (Desktop Only)

Some motherboards have a dedicated CMOS reset jumper:

1. **Power off and unplug** the computer
2. Open the case and locate the **CLRTC**, **CLR_CMOS**, or **JBAT1** jumper on the motherboard
   - Check your motherboard manual or search **"[motherboard model] CMOS jumper location"**
3. The jumper has 3 pins with a cap on pins 1-2 (normal position)
4. Move the cap to pins **2-3** (reset position)
5. Wait **10 seconds**
6. Move the cap back to pins **1-2**
7. Power on

Reference: [Motherboard manual lookup — ASUS](https://www.asus.com/support/), [MSI](https://www.msi.com/support), [Gigabyte](https://www.gigabyte.com/Support)

## Method 4: BIOS Recovery Mode (Corrupted Firmware)

If a BIOS update was interrupted or the firmware is corrupted, many motherboards have a recovery mechanism:

### ASUS — BIOS Flashback

1. Download the latest BIOS from [asus.com/support](https://www.asus.com/support/) for your exact model
2. Rename the file according to the manual (often to a specific name like the board model)
3. Copy it to the root of a FAT32-formatted USB drive
4. Plug the USB into the designated **BIOS Flashback USB port** (usually labeled or colored differently)
5. With the PC powered off but plugged in, press and hold the **BIOS Flashback button** for 3 seconds
6. The LED will flash — wait until it stops (5–10 minutes)
7. Power on

### MSI — M-Flash

1. Download BIOS from [msi.com](https://www.msi.com/support)
2. Copy to FAT32 USB
3. Enter UEFI → M-Flash → Select the file
4. Wait for the update to complete

### Dell — BIOS Recovery

1. Download BIOS from [dell.com/support](https://www.dell.com/support/home)
2. Copy the .exe file to a FAT32 USB
3. Power off, insert USB
4. Hold **Ctrl + Esc** while pressing the power button
5. Release when the recovery screen appears

> **DO NOT** power off or unplug during a BIOS flash. A failed flash can permanently brick the motherboard.

## Restoring Boot Order After Reset

After any BIOS/UEFI reset, your boot order will revert to defaults:

1. Enter UEFI settings
2. Go to the **Boot** tab
3. Set your OS drive as the **first boot device**:
   - Look for "Windows Boot Manager" (Windows) or "ubuntu" / "fedora" (Linux)
   - If you see only drive names, select the drive containing your OS
4. Enable or disable **Secure Boot** as needed:
   - Windows 11: Secure Boot **enabled** (required)
   - Ubuntu/Fedora: Secure Boot **enabled** (supported)
   - Other Linux: Secure Boot **disabled** (may not be signed)
5. Save and exit

## Troubleshooting

| Problem | Solution |
|---|---|
| Still no display after CMOS reset | Try reseating RAM and GPU. Test with one RAM stick at a time. |
| Beep codes on boot | Look up the beep pattern in your motherboard manual — each pattern indicates a specific hardware failure |
| UEFI shows but can't find boot drive | Check SATA/NVMe cables. Ensure the drive appears in the UEFI's storage list. |
| "Secure Boot violation" error | Disable Secure Boot or re-enroll keys (UEFI → Security → Restore Factory Keys) |
| System time keeps resetting | Replace the CMOS battery (CR2032) |

## Common Mistakes

- Interrupting a BIOS update (can permanently brick the board)
- Resetting BIOS on school or work machines without IT approval
- Forgetting to restore boot order after reset
- Not grounding yourself before touching internal components
- Using the wrong BIOS update file (always match your exact model number)
- Running BIOS operations on laptop battery power

## Verification

- [ ] Computer boots to the operating system
- [ ] UEFI settings are accessible at startup
- [ ] Boot order is correct (OS drive first)
- [ ] Date and time are correct (CMOS reset resets the clock)
- [ ] All hardware is detected in the OS (check Device Manager on Windows or `lspci` on Linux)
- [ ] Secure Boot is set correctly for your OS
